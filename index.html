Раздел I
Основные и системные команды

Фундамент любого мира — без этих заклинаний не выстоит ни одна крепость.

Система
/help [команда]
Список доступных команд или справка по конкретной
▼
/help
/help scoreboard

Выводит список всех доступных команд. При указании имени — показывает синтаксис конкретной команды.

Система
/reload
Перезагрузка датапаков, функций, лут-таблиц и рецептов
▼
/reload
⚒ Для датапакеров
Главная команда разработчика. После каждого изменения в файлах датапака — обязательно /reload. Не перезапускает сервер.
Игрок
/gamemode <режим> [игрок]
Смена игрового режима
▼
/gamemode survival
/gamemode creative @a
/gamemode spectator SteveTheWarrior
РежимОписание
survivalВыживание — HP, голод, инвентарь
creativeТворческий — полёт, бесконечные ресурсы
adventureПриключение — только взаимодействие
spectatorНаблюдатель — летает сквозь блоки
Мир
/gamerule <правило> [значение]
Настройка правил мира
▼
/gamerule keepInventory true
/gamerule doDaylightCycle false
/gamerule maxCommandChainLength 65536
/gamerule commandModificationBlockLimit 32768
ПравилоНазначение
keepInventoryСохранять инвентарь после смерти
doDaylightCycleСмена дня и ночи
commandBlockOutputЛогировать вывод КБ в чат
maxCommandChainLengthЛимит команд за тик (датапак)
commandModificationBlockLimitЛимит блоков для /fill и /clone
Мир
/tp <цель> <x y z>
Телепортация сущности в точку или к другой сущности
▼
/tp @s 100 64 -200
/tp @a @p
/teleport @e[type=zombie] ~ ~10 ~ # ~ = текущая координата
Тильда (~) и Каретка (^)
~ — относительные координаты от исполнителя. ^ — локальные координаты (вперёд/вправо/вверх от взгляда).
Мир
/time <set|add|query> <значение>
Управление временем суток
▼
/time set day # 1000 тиков
/time set noon # 6000
/time set night # 13000
/time set 18000 # Полночь
/time add 6000
/time query daytime
Сущность
/kill [цель]
Мгновенное уничтожение сущности
▼
/kill # Убить себя
/kill @e[type=zombie] # Всех зомби
/kill @e[type=!player] # Всё кроме игроков

Раздел II
Селекторы целей

Кого ты призываешь своим заклинанием — определяют руны-селекторы.

@p
Ближайший игрок

Выбирает одного ближайшего к исполнителю живого игрока

@r
Случайный игрок

Выбирает случайного игрока на сервере

@a
Все игроки

Все игроки, включая мёртвых и в режиме наблюдателя

@e
Все сущности

Все живые и технические сущности: мобы, снаряды, маркеры, стойки

@s
Исполнитель

Текущий контекст выполнения — тот, кто запустил команду

Фильтры
Спецификаторы [@...]
Фильтрация и ограничение выборки сущностей
▼
# Только зомби в радиусе 10 блоков
@e[type=zombie,distance=..10]

# Ближайший игрок с тегом и очками
@a[tag=warrior,scores={kills=5..},limit=1,sort=nearest]

# Все, кроме игроков, в прямоугольнике
@e[type=!player,dx=20,dy=10,dz=20]
ПараметрОписание
distance=..10Радиус поиска (..10 = до 10, 5..10 = от 5 до 10)
dx,dy,dzОбъём зоны поиска параллелепипедом
type=zombieТип сущности. ! инвертирует: type=!player
tag=bossНаличие строкового тега. tag=!init — отсутствие
team=RedФильтр по скорборд-команде
name="Steve"Фильтр по имени сущности
scores={deaths=5..}Проверка очков скорборда
level=30..Уровень опыта игрока
gamemode=creativeИгровой режим
limit=1Ограничение числа целей
sort=nearestСортировка: nearest, furthest, random, arbitrary
nbt={OnGround:1b}Проверка NBT-данных (медленно!)
predicate=ns:checkФайл предиката из датапака

Раздел III
/execute — Конструктор контекста

Самое мощное заклинание зала. Строится цепочкой модификаторов — каждый меняет контекст выполнения.

Контекст
Модификаторы контекста
as, at, in, positioned, rotated, facing, anchored
▼
# Выполнить команду от имени каждого игрока
/execute as @a run say Я воин!

# От имени всех зомби, на их позиции
/execute as @e[type=zombie] at @s run particle flame ~ ~ ~

# Смотреть в сторону ближайшего игрока
/execute as @e[tag=guard] at @s facing entity @p eyes run ...
МодификаторЧто меняет
as <цель>Исполнителя (@s). Координаты НЕ меняет
at <цель>Позицию, поворот и измерение на принадлежащие цели
in <измерение>Переносит контекст в другое измерение (overworld/nether/end)
positioned <x y z>Только координаты выполнения
rotated <yaw pitch>Только направление взгляда
facing entity @p eyesПовернуть взгляд к глазам/ногам сущности
anchored eyes|feetСмещает ^ ^ ^ относительно уровня глаз или ног
Условия
if / unless — Ветвление
Условные операторы внутри /execute
▼
# Если под ногами камень — дать эффект
/execute as @a at @s if block ~ ~-1 ~ stone run effect give @s speed 1 0

# Если в мире есть дракон
/execute if entity @e[type=ender_dragon] run say Дракон жив!

# Если очки kills >= 10
/execute as @a if score @s kills matches 10.. run ...

# Сложная цепочка (пример из документа)
/execute as @a[tag=active] at @s anchored eyes if block ^ ^ ^1 air run particle flame ^ ^ ^1
УсловиеОписание
if block <xyz> <блок>Проверка конкретного блока по координатам
if blocks r1 r2 dest all|maskedСравнение двух регионов блоков
if entity <сущность>Существует ли хотя бы одна такая сущность
if score ... matchesСравнение очков скорборда (< <= = > >= matches)
if data <block|entity|storage>Существует ли NBT-тег по пути
if predicate <id>Запуск файла-предиката из датапака
if dimension <id>Контекст находится в указанном измерении?
if loaded <xyz>Прогружен ли чанк по координатам (критично!)
Хранение
store result|success
Сохранение результата выполнения в переменную или NBT
▼
# Записать HP игрока в скорборд
/execute store result score @s hp_obj run data get entity @s Health

# Успех (1 или 0) в storage
/execute store success storage my_pack:main result byte 1 run ...
ТипОписание
resultЧисловое значение (кол-во предметов, HP, уровень сигнала)
success1 если команда выполнена, 0 если нет
score <цель> <объект>В переменную скорборда
storage <id> <путь> <тип> <x>В глобальное хранилище датапака
entity <сущность> <путь>В NBT сущности
block <xyz> <путь>В NBT блока (сундук, КБ)
bossbar <id> value|maxВ значение боссбара
Типы данных
byte · short · int · long · float · double

Раздел IV
/scoreboard — Игровая СУБД

Система переменных и очков. Хранит числа, отслеживает статистику, создаёт игровые механики.

Objectives
scoreboard objectives
Создание и управление задачами (переменными)
▼
/scoreboard objectives add kills playerKillCount
/scoreboard objectives add timer dummy "Таймер"
/scoreboard objectives setdisplay sidebar kills
/scoreboard objectives list
/scoreboard objectives remove timer
КритерийЧто отслеживает
dummyИзменяется только командами (основной для датапаков)
deathCountКоличество смертей игрока
playerKillCountУбийства игроков
custom:minecraft.leave_gameВыходы из игры
Ячейки отображения
sidebar · list · belowName · sidebar.team.red
Players
scoreboard players
Работа с очками сущностей и математика
▼
/scoreboard players set @s timer 100
/scoreboard players add @s kills 1
/scoreboard players remove @a hp 5
/scoreboard players get @s kills
/scoreboard players reset @s timer

# Математика между переменными
/scoreboard players operation @s result += @s bonus
/scoreboard players operation @s hp %= #const divider
/scoreboard players operation @s a >< @s b # Обмен
Операции
= · += · -= · *= · /= · %= · > (макс) · < (мин) · >< (обмен)
Фейковые ники
Используй #global или #const как имена без реального игрока: scoreboard players set #global timer 0

Раздел V
/data — Манипуляция NBT

Прямое чтение и запись в тайные хранилища данных — сущностей, блоков и глобального storage.

NBT
/data get|merge|modify|remove
Читать и изменять NBT у сущностей, блоков и хранилищ
▼
# Читать HP игрока
/data get entity @s Health

# Слить данные с сущностью
/data merge entity @s {CustomName:'{"text":"Воин"}'}

# Добавить в конец массива хранилища
/data modify storage my_pack:main cooldowns append value 100

# Скопировать из сундука в storage
/data modify storage my_pack:main chest_data set from block ~ ~ ~ Items

# Удалить тег
/data remove entity @s CustomName
КомандаОписание
/data getВывод NBT или значения тега (с множителем)
/data mergeСлияние — перезаписывает совпадения, добавляет новые
/data removeУдаление тега по пути
/data modify ... setПолная замена значения
/data modify ... appendДобавить в конец списка
/data modify ... prependДобавить в начало списка
/data modify ... insert NВставить по индексу
⚠ Игрокам нельзя менять NBT напрямую
Только читать. /data merge entity @s не работает для игрока. Используй /attribute или компоненты.

Раздел VI
/item + Компоненты предметов

В версии 1.20.5+ старые NBT-теги заменены системой компонентов в квадратных скобках.

Инвентарь
/give <цель> <предмет[компоненты]>
Выдача предмета с кастомными свойствами
▼
# Меч с зачарованием и кастомным именем
/give @s netherite_sword[
custom_name='{"text":"Клинок Судьбы","color":"gold","bold":true}',
enchantments={levels:{"minecraft:sharpness":5,"minecraft:looting":3}},
lore=['{"text":"Выкован в горниле","color":"gray"}'],
unbreakable={}]

# Кастомная еда
/give @s iron_ingot[food={nutrition:4,saturation:2.5,can_always_eat:true}]
КомпонентОписание
custom_data={...}Кастомные NBT-теги для датапаков
custom_name='...'Имя предмета в JSON-формате
lore=[...]Описание — массив JSON-строк
damage=100Текущая поломка инструмента
max_stack_size=99Максимальный размер стака (1–99)
enchantment_glint_override=trueПринудительный блеск зачарования
enchantments={levels:{...}}Чары и их уровни
attribute_modifiers=[...]Кастомные атрибуты (скорость, урон)
food={nutrition:4,...}Сделать предмет съедобным
unbreakable={}Неломаемый инструмент
Инвентарь
/item replace|modify
Замена и изменение предметов в слотах
▼
# Заменить предмет в основной руке
/item replace entity @s weapon.mainhand with diamond_sword

# Первый слот сундука
/item replace block ~ ~ ~ container.0 with apple 32

# Применить модификатор из датапака
/item modify entity @s weapon.mainhand my_pack:add_fire
Чары
/enchant <цель> <чар> [уровень]
Зачаровать предмет в руке
▼
/enchant @s minecraft:sharpness 5
/enchant @a minecraft:protection 4
Ограничение выживания
Работает с учётом ограничений режима выживания. Для снятия ограничений — используй /give с компонентом enchantments.
Инвентарь
/clear [цель] [предмет] [кол-во]
Очистка инвентаря
▼
/clear # Очистить себя полностью
/clear @a # Всех игроков
/clear @s minecraft:dirt # Только землю
/clear @s minecraft:dirt 32 # Удалить 32 блока земли

Раздел VII
Мир и блоки

Переписать облик мира — setblock, fill, clone, fillbiome и структуры.

Блоки
/setblock <x y z> <блок> [режим]
Установить один блок по координатам
▼
/setblock 100 64 -200 minecraft:stone
/setblock ~ ~-1 ~ minecraft:chest[facing=north] replace
/setblock ~ ~ ~ minecraft:air destroy
РежимОписание
destroyЛомает блок с частицами и дропом предметов
keepСтавит только если там был воздух
replaceМолча заменяет (по умолчанию)
Блоки
/fill <x1 y1 z1> <x2 y2 z2> <блок> [режим]
Заполнение области блоками
▼
/fill ~-5 ~-1 ~-5 ~5 ~5 ~5 minecraft:stone
/fill 0 60 0 20 70 20 minecraft:oak_planks hollow
/fill 0 64 0 100 64 100 minecraft:grass_block replace minecraft:dirt
РежимОписание
hollowКоробка с воздухом внутри
outlineКоробка, не трогает содержимое
keepЗаполняет только воздух
replace [фильтр]Заменяет только указанный тип блоков
destroyЛомает с дропом (медленно!)
Блоки
/clone <r1> <r2> <dest> [режим]
Копирование области блоков в другое место
▼
/clone 0 64 0 10 74 10 100 64 100 masked
/clone 0 64 0 10 74 10 50 64 50 filtered minecraft:chest
/clone 0 64 0 10 74 10 0 64 15 replace move
РежимОписание
maskedИгнорирует воздух источника
filtered <блок>Копирует только указанный тип
moveПереносит, удаляет на старом месте
forceКопирование с перекрытием зон
Структуры
/place structure|feature|template
Генерация структур и элементов мира
▼
# Деревня
/place structure minecraft:village_snowy ~ ~ ~

# Одно дерево
/place feature minecraft:oak ~ ~ ~

# Файл .nbt из датапака (полуразрушенный на 70%)
/place template my_pack:my_building ~ ~ ~ none none 0.7 12345
integrity
Параметр 0.0–1.0 для /place template: спавн структуры сразу полуразрушенной. 1.0 = целая, 0.5 = половина блоков.
Биомы
/fillbiome <r1> <r2> <id_биома>
Изменение биома в регионе
▼
/fillbiome 0 -64 0 100 320 100 minecraft:desert
⚠ Требует перегрузки чанков
Нажми F3+A после выполнения, чтобы увидеть смену цвета травы и воды. На сервере каждый игрок должен сделать это сам.

Сущности

# Дисплей блока

Техническая сущностьНазначение
minecraft:markerНевидимая точка, не лагает. Для хранения координат и тегов
minecraft:interactionНевидимый хитбокс — считывает клики и атаки игроков

minecraft:item_displayРендеринг предметов под любым углом
Эффекты

Выдача и снятие эффектов зелий
▼

# длит.(сек) уровень скрыть_частицы

/effect clear @s # Снять всё
/effect clear @s minecraft:poison # Только яд
Атрибуты
/attribute <цель> <атрибут> <get|base|modifier>
Изменение базовых параметров сущностей
▼

/attribute @s generic.scale base set 2.0 # 1.20.5+
/attribute @s generic.gravity base set 0.04
АтрибутОписание
generic.max_healthМаксимальное здоровье
generic.movement_speedСкорость ходьбы (0.1 = норма)
generic.attack_damageБазовый урон
generic.scaleФизический масштаб (0.0625–16.0), 1.20.5+
generic.step_heightВысота блока без прыжка
generic.gravityСила гравитации
generic.jump_strengthСила прыжка

Управление опытом игрока
▼

Сервер
Команды администратора

/op Steve # Выдать права оператора

Дебаг
/perf <start|stop> и /debug
Профилирование производительности сервера
▼
/perf start # Запустить 10-20 сек мониторинг
/perf stop # Отчёт в /debug/profiler/

Когда использовать
/perf незаменим при падении TPS. Запусти на 15 секунд нагрузки, остановить — и читай отчёт: он покажет, какие функции датапака или мобы пожирают тики.

Раздел X
/tick — Управление тикрейтом

Контроль над самим временем мира — заморозить, ускорить, прокрутить покадрово.

Движок
/tick query|rate|freeze|step|sprint
Полный контроль скорости игрового времени
▼
/tick query # Текущий MSPT и TPS
/tick rate 20 # Норма (20 тиков/сек)
/tick rate 5 # Замедление в 4 раза
/tick rate 500 # Ускорение для теста ферм
/tick freeze # Заморозить мир
/tick step 1 # Прокрутить 1 тик (при freeze)
/tick step 100 # 100 тиков вперёд
/tick unfreeze # Снять паузу
/tick sprint 72000 # Прокрутить час игрового времени
⚒ Для датапакеров
/tick freeze + /tick step 1 = покадровый дебаг. Можно остановить мир, проверить состояние всех NBT и очков, и проверить следующий тик изолированно.

Раздел XI
Прочие команды
Датапак
/function <имя> и /return
Запуск файлов функций и возврат значения
▼
/function my_pack:init
/function my_pack:boss/phase_2

# Макрос (1.20.2+) — передача аргументов
/function my_pack:set_hp {value: 20}
# В файле функции: /attribute @s generic.max_health base set $(value)

# Возврат значения из функции
/return 1
Датапак
/tag <цель> <add|remove|list> <имя>
Строковые метки на сущностях для фильтрации
▼
/tag @s add warrior
/tag @e[type=zombie] add custom_boss
/tag @s remove init
# Использование: @e[tag=custom_boss,limit=1]
UI
/bossbar
Создание и управление полосками боссов
▼
/bossbar add my_pack:raid '{"text":"Набег"}'
/bossbar set my_pack:raid color red
/bossbar set my_pack:raid style notched_10
/bossbar set my_pack:raid max 100
/bossbar set my_pack:raid value 75
/bossbar set my_pack:raid players @a
Цвета
pink · blue · red · green · yellow · purple · white
Стили
progress · notched_6 · notched_10 · notched_12 · notched_20
UI
/title <цель> <title|subtitle|actionbar>
Вывод текста на экран игрока
▼
/title @a title '{"text":"ВЫ ПОБЕДИЛИ","color":"gold","bold":true}'
/title @a subtitle '{"text":"Волна завершена","color":"gray"}'
/title @s actionbar '{"text":"Критический удар!","color":"red"}'
/title @a times 10 40 10 # fade_in stay fade_out (в тиках)
/title @a clear
Игрок
/advancement grant|revoke
Выдача и отзыв достижений
▼
/advancement grant @s only my_pack:first_kill
/advancement grant @s everything
/advancement revoke @s from minecraft:story/root
В датапаках
Достижения с trigger=impossible используются как невидимые события — игрок не видит их в меню, но /execute if score срабатывает.
Мир
/worldborder
Управление границей мира
▼
/worldborder set 1000 # Радиус 1000 блоков
/worldborder add -100 60 # Сужать на 100 блоков за 60 сек
/worldborder center 0 0
/worldborder damage amount 0.5
/worldborder warning distance 50
